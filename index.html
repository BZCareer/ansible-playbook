Monitor.d

Monitor Your Kubernetes Cluster 100% Open Source
Founded By Zak Hassan

What is the motivation for Monitor.d ?

When doing performance testing. I don't want to put agents or spit out logs on disk and run into out of disk space errors. In docker environments using kubernetes I would like to have a dashboard that will give me information about: Network, IO, Disk space, Memory, Running Services, and ability to audit events like changes to logs, and deployments.

What platform will Monitor.d run on ?

It will run on openshift/origin which will be on top of kubernetes, running docker.
